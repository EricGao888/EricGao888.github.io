General Solution for DP

  1. Consider all the possible strategies or conditions in the $Cth$ step;
  2. Treat previous state as known, for every different possible strategies or conditions(assume there are $k$ different possible strategies or conditions) find the corresponding previous state and get the $k$ results on the $Cth$ step;
  3. Compare all the $k$ results and determine the best stragegy for the $Cth$ strp;
  • You must consider all possible strategies or condtions on the $Cth$ Step!

Understanding DP

  • Don’t think too much about global optimal and local optimal. Different from Greedy Algo, DP is a kind of State Machine. Greedy takes local optimal at every step and that corresponds to the global optimal. However, in many cases, taking local optimal at each step will not lead to global optimal. DP memorizes the previous state and decide the current state based on combinations and modifications of previous state;
  • In DP we ofte see the initialization of meaningless state(before start or after end) as zero;

Shortest Path

  • Dijkstra: 基于贪心，在此每步取局部最优最终能达到全局最优。每次新扩展一个路程最短的点，更新与其相邻的点点路程。当所有遍权都为正时，由于不会存在一个路程更短的没扩展过的点，所以这个点的路程永远不会再改变，因而保证了算法的正确性。因此次算法不适用于含负权边的图，否则会破坏已经更新的点路程不会改变的性质；

Recursion

  • Difference between recursion and iteration;
  • 在使用递归时；必须有一个明确的递归结束条件；称为递归出口;
  • 递归由递推和回归两部分组成，递推是把复杂的问题的求解推到比原问题简单一些的问题的求解；回归是当获得最简单的情况后，逐步返回，依次得到复杂的解；

LeetCode Tutorial

  • Two ways to construct Hash Table - Separate Chaining and Linear Probing
  • There are two different kinds of hash tables: hash set and hash map;
  • Typically, if N is constant and small, we can simply use an array to store keys in the same bucket. If N is variable or large, we might need to use height-balanced binary search tree instead;
  • Insertion and search are two basic operations in a hash table. Besides, there are operations which are based on these two operations. For example, when we remove an element, we will first search the element and then remove the element from the corresponding position if the element exists.

Delete Elements in Hash Table Implemented by Linear Probing

  • The easiest way to implement delete is to find and remove the key–value pair and then to reinsert all of the key–value pairs in the same cluster that appear after the deleted key–value pair. If the hash table doesn’t get too full, the expected number of key–value pairs to reinsert will be a small constant.
  • An alternative is to flag the deleted linear-probing table entry so that it is skipped over during a search but is used for an insertion. If there are too many flagged entries, create a new hash table and rehash all key–value pairs.

Complexity Analysis

  • Time: Look up an element is O(1), but could degenerate to O(n);
  • Space: O(n) if the map stores n elements;
  • The space complexity of a hash table:

The hash function is computed modulo the size of a reference vector that is much smaller than the hash function range. Because this value is fixed, it is not considered in the space complexity computation. Consequently, the space complexity of every reasonable hash table is O(n).
